  SECTION 1: Merge

  Q1. What is the difference between on= and left_on= / right_on= in .merge()?

  A: Use on= when both DataFrames share the same column name. Use left_on= / right_on= when the key columns have
  different names in each DataFrame.
  # same name
  report.merge(currency_df, on='currency')

  # different names
  report.merge(currency_df, left_on='currency', right_on='code')

  ---
  Q2. What does how='left' do in a merge, and when would you use it?

  A: A left join keeps all rows from the left DataFrame. Rows with no match in the right DataFrame get NaN for
  the right side's columns. Use it when you want to preserve all source rows even if lookup data is incomplete.

  ---
  Q3. What does validate='many_to_one' do, and why is it useful?

  A: It tells pandas to assert that keys in the right DataFrame are unique. If a duplicate key is found, it
  raises an error. Without it, a merge can silently multiply rows — a hard-to-catch bug.

  ---
  Q4. How do you merge on multiple columns?

  A: Pass a list to on=:
  df.merge(stats, on=['currency', 'validation_status'])
  Or if names differ:
  df.merge(stats, left_on=['currency', 'status'], right_on=['cur', 'val_status'])

  ---
  Q5. What are merge suffixes and when do they appear?

  A: When both DataFrames have a column with the same name (other than the key), pandas adds _x and _y suffixes
  by default. You can customize them:
  df.merge(other, on='id', suffixes=('', '_by_currency'))
  The best practice is to rename columns before merging to avoid the problem entirely.

  ---
  SECTION 2: Explode, Concat

  Q6. What does .explode() do?

  A: It takes a column containing lists and expands each list element into its own row, duplicating the other
  column values. Used for one-to-many transformations.
  df.assign(error_code=df['error_codes'].str.split(', ')).explode('error_code')

  ---
  Q7. What is the difference between pd.concat() with and without ignore_index=True?

  A: Without it, the original index values are preserved, which can create duplicate index values. With
  ignore_index=True, pandas assigns a fresh 0-based index across the combined DataFrame.

  ---
  Q8. How do you track which source each row came from after pd.concat()?

  A: Add a tag column before concatenating:
  clean_tagged = clean.assign(source='clean')
  flagged_tagged = flagged.assign(source='flagged')
  pd.concat([clean_tagged, flagged_tagged], ignore_index=True)

  SECTION 3: Pivot Table & Melt

  Q9. What are the four main parameters of pd.pivot_table()?

  A:

  ┌───────────┬─────────────────────────────────────────┐
  │ Parameter │                  Role                   │
  ├───────────┼─────────────────────────────────────────┤
  │ values    │ Column to aggregate                     │
  ├───────────┼─────────────────────────────────────────┤
  │ index     │ Becomes the row labels                  │
  ├───────────┼─────────────────────────────────────────┤
  │ columns   │ Unique values become column headers     │
  ├───────────┼─────────────────────────────────────────┤
  │ aggfunc   │ How to aggregate (e.g. 'count', 'mean') │
  └───────────┴─────────────────────────────────────────┘

  ---
  Q10. What does margins=True add to a pivot table?

  A: It adds a totals row and column. Use margins_name='Overall' to rename them from the default 'All'.

  ---
  Q11. What is pd.melt() and what are its key parameters?

  A: melt() converts wide format to long format — it "unpivots" columns into rows.
  - id_vars: columns to keep as-is (identifiers)
  - value_vars: columns to stack into rows
  - var_name: name for the new "column name" column
  - value_name: name for the new "value" column

  ---
  Q12. How many rows does melt produce if you melt 2 columns from a 200-row DataFrame?

  A: 400 rows — each row becomes 2 rows (one per value_var).

  ---
  SECTION 4: Stack / Unstack / MultiIndex

  Q13. What is the difference between .stack() and .unstack()?

  A: .stack() moves columns into an index level (wide → long). .unstack() moves an index level back to columns
  (long → wide). They are inverses of each other.

  ---
  Q14. How do you select all rows for one top-level group in a MultiIndex?

  A: Use .loc[] with just the outer level key:
  multi.loc['EUR']           # all validation statuses for EUR
  multi.loc[('EUR', 'FLAGGED')]  # specific cell — needs tuple

  ---
  Q15. What does .xs() do and when is it preferred over .loc[]?

  A: .xs() (cross-section) lets you slice on any level by name, not just the outermost one:
  multi.xs('FLAGGED', level='validation_status')  # all currencies that are FLAGGED
  Use it when you want to filter by an inner level without specifying the outer level first.

  ---
  Q16. How do you flatten MultiIndex columns after using aggfunc=['count', 'mean']?

  A:
  pivot.columns = ['_'.join(col) for col in pivot.columns]
  # ('count', 'CLEAN') → 'count_CLEAN'

  ---
  SECTION 5: Map, Apply, np.where, np.select

  Q17. What is the difference between .map() and .apply()?

  A:
  - .map() is for simple, element-wise transforms on a Series: dict lookup, built-in function, or simple lambda.
  - .apply() is more flexible and can handle complex logic. With axis=1 it passes an entire row to the function.

  ---
  Q18. When would you use np.where() instead of .apply(axis=1)?

  A: Use np.where() when the logic is a simple if/else on column values — it's vectorized (runs at C speed) and
  can be 10–100x faster than apply. Use apply only when logic is too complex for vectorized operations.

  ---
  Q19. How does np.select() work?

  A: It's vectorized multi-branch logic (like if/elif/elif/else):
  conditions = [df['x'] == 0, df['x'] <= 2, df['x'] <= 4]
  choices    = ['no_risk',    'low_risk',   'medium_risk']
  df['risk'] = np.select(conditions, choices, default='high_risk')
  First matching condition wins.

  ---
  Q20. What does axis=1 mean in .apply(func, axis=1)?

  A: It sends each row (as a Series) to func. Default axis=0 sends each column. Think: axis=1 → working
  horizontally across columns.

  try:

          bad_lookup,
          on='error_code',
          how='left',

      )

      print(f"\nWith validate='many_to_one' error raised:")
      print(type(e).__name__, '—', e)

  What to notice: Without validate, the merge silently inflates rows — a bug that is very hard to spot downstream. With validate='many_to_one', pandas
  catches it immediately and raises a MergeError. Always use validate when you know the lookup should be unique.
