  Merging DataFrames                                                                                                                                                            
                                                                                                                                                                                
  Q1: What does pd.merge() do by default, and why is that dangerous?                                                                                                            
                                                                                                                                                                                
  By default, merge uses how='inner', which silently drops rows that don't match. Example:                                                                                      
  # If error_code 'E999' is not in error_lookup → that row is GONE with no warning                                                                                              
  errors_enriched = errors_exploded.merge(error_lookup, on='error_code')                                                                                                        
  # 237 rows in → only 45 out (192 dropped silently!)                                                                                                                           
                                                                                                                                                                                
  # Safe version: use how='left' — keeps all rows, unmatched get NaN                                                                                                            
  errors_enriched = errors_exploded.merge(error_lookup, on='error_code', how='left')                                                                                            
  # 237 rows in → 237 rows out                                                                                                                                                  
  Always specify how= explicitly.                                                                                                                                               
                  
  ---                                                                                                                                                                           
  Q2: When do you use left_on= / right_on= instead of on=?
                                                                                                                                                                                
  When the key columns have different names in each DataFrame:
  # 'currency' in report, but 'code' in currency_df — different names                                                                                                           
  merged = report.merge(currency_df, left_on='currency', right_on='code')
                                                                                                                                                                                
  # Same name in both → just use on=                                                                                                                                            
  merged = errors_exploded.merge(error_lookup, on='error_code')                                                                                                                 
                                                                                                                                                                                
  ---                                                                                                                                                                           
  Q3: What does validate='many_to_one' do in a merge?
                                                                                                                                                                                
  It checks that the right DataFrame has no duplicate keys — meaning each key appears only once on the right side. This catches bugs before they silently multiply rows:
  merged = errors_exploded.merge(                                                                                                                                               
      error_lookup,              
      on='error_code',                                                                                                                                                          
      how='left',                                                                                                                                                               
      validate='many_to_one'  # Raises MergeError if error_lookup has duplicate error_codes
  )                                                                                                                                                                             
  Use this for lookup merges. 'many_to_many' never validates — avoid it.                                                                                                        
   
  ---                                                                                                                                                                           
  Q4: When do you use .map(dict) vs .merge()?
                                                                                                                                                                                
  ┌──────────────┬──────────────────────┬─────────────────────────────────┐
  │              │      .map(dict)      │            .merge()             │                                                                                                     
  ├──────────────┼──────────────────────┼─────────────────────────────────┤
  │ Source       │ Python dict          │ DataFrame                       │                                                                                                     
  ├──────────────┼──────────────────────┼─────────────────────────────────┤
  │ Adds         │ One column at a time │ Many columns at once            │
  ├──────────────┼──────────────────────┼─────────────────────────────────┤                                                                                                     
  │ Missing keys │ Returns NaN          │ Drops row (inner) or NaN (left) │
  └──────────────┴──────────────────────┴─────────────────────────────────┘                                                                                                     
                  
  # map: quick 1-column lookup from a plain dict                                                                                                                                
  report['currency_symbol'] = report['currency'].map({'USD': '$', 'EUR': '€'})                                                                                                  
   
  # merge: pull in multiple columns from a DataFrame lookup table                                                                                                               
  report.merge(currency_df, left_on='currency', right_on='code')
  # Adds: name, symbol, decimals all at once                                                                                                                                    
                  
  ---                                                                                                                                                                           
  Q5: How do you merge on multiple columns?
                                                                                                                                                                                
  Pass a list to on=:
  # Both 'currency' AND 'validation_status' must match                                                                                                                          
  enriched = report.merge(                            
      currency_status_stats,                                                                                                                                                    
      on=['currency', 'validation_status'],                                                                                                                                     
      how='left',                          
      validate='many_to_one'                                                                                                                                                    
  )

  Q6: What happens when two DataFrames have a column with the same name (that's NOT the key)?                                                                                   
                                                                                                                                                                                
  Pandas adds _x and _y suffixes automatically. Better approach: rename before merging:
  # Instead of getting avg_errors_x and avg_errors_y:                                                                                                                           
  by_currency = by_currency.rename(columns={'error_count': 'avg_errors_by_currency'})
  by_status   = by_status.rename(columns={'error_count': 'avg_errors_by_status'})                                                                                               
                                                                                 
  # Now chain merges cleanly                                                                                                                                                    
  result = (report                                                                                                                                                              
      .merge(by_currency, on='currency', how='left')                                                                                                                            
      .merge(by_status, on='validation_status', how='left'))

  Concat
                                                                                                                                                                                
  Q7: What is the difference between merge() and concat()?

  ┌─────────────┬───────────────────────────┬──────────────────────────────┐                                                                                                    
  │             │          merge()          │           concat()           │
  ├─────────────┼───────────────────────────┼──────────────────────────────┤                                                                                                    
  │ Direction   │ Horizontal — adds columns │ Vertical — adds rows         │
  ├─────────────┼───────────────────────────┼──────────────────────────────┤
  │ Purpose     │ Combine by matching key   │ Stack same-shaped DataFrames │
  ├─────────────┼───────────────────────────┼──────────────────────────────┤                                                                                                    
  │ Key needed? │ Yes                       │ No                           │
  └─────────────┴───────────────────────────┴──────────────────────────────┘                                                                                                    
                  
  # merge: "I have extra info about my existing rows"
  report.merge(currency_df, left_on='currency', right_on='code')
                                                                                                                                                                                
  # concat: "I have more rows of the same type"
  pd.concat([clean, flagged], ignore_index=True)                                                                                                                                
                  
  ---
  Q8: What does ignore_index=True do in pd.concat()?
                                                                                                                                                                                
  It resets the index to 0, 1, 2, ... instead of keeping the original indices (which could cause duplicates):
  # Without: index repeats (0–71 from clean, 0–127 from flagged)                                                                                                                
  pd.concat([clean, flagged])                                   
                                                                                                                                                                                
  # With: fresh clean index 0–199
  pd.concat([clean, flagged], ignore_index=True)                                                                                                                                
                  
  ---                                                                                                                                                                           
  Q9: How do you add a source tag when concatenating so you know where each row came from?
                                                                                                                                                                                
  Use .assign() before concatenating:
  clean_tagged   = clean.assign(source='clean')                                                                                                                                 
  flagged_tagged = flagged.assign(source='flagged')
                                                                                                                                                                                
  recombined = pd.concat([clean_tagged, flagged_tagged], ignore_index=True)                                                                                                     
  # Now recombined['source'] tells you the origin of every row

  Pivot Tables
                                                                                                                                                                                
  Q10: What are the four parameters of pd.pivot_table() and what does each do?
                                                                                                                                                                                
  pd.pivot_table(
      report,                                                                                                                                                                   
      values='error_count',         # WHAT data to aggregate
      index='currency',             # WHAT goes on ROWS                                                                                                                         
      columns='validation_status',  # WHAT goes across COLUMNS                                                                                                                  
      aggfunc='mean'                # HOW to combine duplicates                                                                                                                 
  )                                                                                                                                                                             
  Mental model: it's an Excel pivot table — drag fields into rows, columns, values, and pick an aggregation.
                                                                                                                                                                                
  ---             
  Q11: How do you add totals (grand total row/column) to a pivot table?                                                                                                         
                                                                                                                                                                                
  pd.pivot_table(
      report,                                                                                                                                                                   
      values='error_count',
      index='currency',                                                                                                                                                         
      columns='validation_status',
      aggfunc='mean',                                                                                                                                                           
      margins=True,           # Adds totals
      margins_name='Overall'  # Names the totals row/column                                                                                                                     
  ).round(2)
  The bottom-right corner value is the overall average across the entire dataset.                                                                                               
                                                                                                                                                                                
  ---
  Q12: How do you use multiple aggregation functions in a pivot table, and how do you flatten the resulting column names?                                                       
                                                                                                                                                                                
  pivot = pd.pivot_table(
      report,                                                                                                                                                                   
      values='error_count',
      index='currency',
      columns='validation_status',
      aggfunc=['count', 'mean']   # Creates MultiIndex columns
  ).round(2)                                                                                                                                                                    
   
  # Columns look like: ('count', 'CLEAN'), ('mean', 'FLAGGED'), etc.                                                                                                            
  # Flatten them: 
  pivot.columns = ['_'.join(col) for col in pivot.columns]                                                                                                                      
  # Result: count_CLEAN, count_FLAGGED, mean_CLEAN, mean_FLAGGED

  Melt (Wide → Long)                                                                                                                                                            
                                                                                                                                                                                
  Q13: What does pd.melt() do, and when would you use it?
                                                                                                                                                                                
  It converts wide format (many columns) to long format (one column with all values stacked). Use it when multiple columns represent the same type of data:                     
  # Wide: 200 rows × 2 country columns
  # Long: 400 rows × 1 country column                                                                                                                                           
                                                                                                                                                                                
  countries_long = pd.melt(
      report,                                                                                                                                                                   
      id_vars=['transaction_id'],                           # Keep as identifier
      value_vars=['applicant_country', 'beneficiary_country'],  # Stack these                                                                                                   
      var_name='party_role',                                # Name for the variable column                                                                                      
      value_name='country'                                  # Name for the values column  
  )                                                                                                                                                                             
  Use melt when you want to analyze/group across columns that represent the same concept.

  Q14: After groupby().agg(), why do the grouped columns disappear, and how do you get them back?
                                                                                                                                                                                
  After groupby(), the grouped columns become the index (not regular columns), so you can't use them in a merge. Use .reset_index() to pull them back:
  # After groupby — currency and validation_status are the INDEX:                                                                                                               
  stats = report.groupby(['currency', 'validation_status']).agg(...)
                                                                                                                                                                                
  # After reset_index() — they're regular columns again:                                                                                                                        
  stats = stats.reset_index()                           
                                                                                                                                                                                
  # Now you can merge on them:                                                                                                                                                  
  report.merge(stats, on=['currency', 'validation_status'], how='left')
  You'll use reset_index() constantly after groupby().

  Q15: What's the difference between wide and long format? When do you use each?
                                                                                                                                                                                
  ┌────────────────┬─────────────────────────────────┬──────────────────────────┐
  │                │              Wide               │           Long           │
  ├────────────────┼─────────────────────────────────┼──────────────────────────┤
  │ Format         │ Each category = separate column │ All values in one column │
  ├────────────────┼─────────────────────────────────┼──────────────────────────┤
  │ Human-readable │ Yes — easy to scan              │ Less so                  │                                                                                               
  ├────────────────┼─────────────────────────────────┼──────────────────────────┤                                                                                               
  │ Programmatic   │ Hard to filter/group            │ Easy to filter/group     │                                                                                               
  ├────────────────┼─────────────────────────────────┼──────────────────────────┤                                                                                               
  │ Created by     │ pivot_table()                   │ groupby() or melt()      │
  └────────────────┴─────────────────────────────────┴──────────────────────────┘                                                                                               
                  
  Typical flow: raw data (long) → groupby() → analyze → pivot_table() → compare visually.
