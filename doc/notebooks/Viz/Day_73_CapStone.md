Decoding the new techniques:

| Piece                          | What it does                              |
|--------------------------------|-------------------------------------------|
| fig.suptitle(..., y=0.98)      | Push title higher (default 0.95 too low)  |
| fig.text(0.5, 0.94, ...)       | Free-form text at FIGURE coords (not axes)|
| ha='center'                    | Center text on the x coordinate           |
| 0.5, 0.94                      | Figure-fraction coords: 50% across, 94% up|
| style='italic'                 | Italic text — secondary visual weight     |
| rect=[0, 0.04, 1, 0.92]        | Layout box: bottom 4% for footer, top 8%  |
|                                | for suptitle+subtitle                      |

The fig.text() trick is what enables subtitles AND footers in matplotlib.
Subplots and titles all live INSIDE axes. fig.text() lives OUTSIDE axes,
in figure-fraction coordinate space (0,0) bottom-left to (1,1) top-right.
That's why we use 0.5 for "center horizontally" — half the figure width.

 * plt.tight_layout(rect=[0, 0.04, 1, 0.92]) -> " Format is [left, bottom, right, top] 
                                           
  Normally tight_layout() uses the entire figure area. The rect parameter tells it: "only lay out the subplots
  inside this box." Format is [left, bottom, right, top] in figure coordinates.                                  
   
  Here: leave the bottom 4% for the footer, leave the top 8% for the suptitle + subtitle. Everything outside that
   rect is free for fig.text() to use.

  hygiene. Any chart sent to a correspondent bank, auditor, or ICC examiner needs to show where the data came    
  from. Without it, the chart is an opinion. With it, it's a traceable finding.

Pro tip — pie chart label visibility issues

Pie charts default to drawing labels OUTSIDE each slice. For very SMALL
slices (e.g., 5%), the label can overlap with neighboring labels.

Two fixes if you ever hit this:
- labeldistance=1.1   # Push outside labels further out (1.0 = default)
- pctdistance=0.75    # Pull percentage labels closer to center

For your 64/36 split, labels are far apart enough that defaults work fine.
But if you ever build a 5-slice pie with tiny percentages, these are the
parameters that save you.

Bigger principle: pie charts work for 2-3 large categories. For 5+
categories, switch to a horizontal bar chart. The eye can't compare
angular slices well at small sizes — that's the well-known "pie chart
anti-pattern" Tufte rants about.

  ---
  Line by line — with the WHY
                                                                                                                 
  donut_colors = [PIPELINE_PALETTE['flagged'], PIPELINE_PALETTE['clean']]
                                                                                                                 
  You could have written ['#DC143C', '#28A745'] directly. Why not?                                               
                                                                                                                 
  Because status_counts.index comes back as ['FLAGGED', 'CLEAN'] — FLAGGED first because it has 128 rows and     
  CLEAN only 72, and .value_counts() sorts descending. The semantic names make it obvious which color maps to
  which slice. If you swap the order someday, you won't accidentally put green on FLAGGED.                       
                  
  ---
  autopct='%1.0f%%'
                   
  This is a mini format string. Unpacked:
                                                                                                                 
  | Part   | Meaning                                      |
  |--------|----------------------------------------------|                                                      
  | %1.0f  | float, minimum 1 char wide, 0 decimal places |
  | %%     | literal % sign (needs escaping in % format)  |                                                      
                                                                                                                 
  So 64.0 → "64%". Without 0f it would show "64.0%" — fine in a notebook, embarrassing in a board report.        
                                                                                                                 
  ---                                                                                                            
  startangle=90   
               
  Matplotlib's default start position is 3 o'clock (math convention: 0° on the right). startangle=90 rotates to
  12 o'clock, which is the convention everywhere — compliance dashboards, Bloomberg terminals, PowerPoint        
  default. Nobody starts a pie chart at 3 o'clock on purpose.
                                                                                                                 
  ---             
  wedgeprops={'width': 0.4} — THE donut trick
                                                                                                                 
  A regular pie has width=1.0 — each wedge fills the full radius. Setting width=0.4 means the wedge only occupies
   the outer 40%, leaving a hole in the center. That's it. That's the entire donut.                              
                  
  | width | Result                       |                                                                       
  |-------|------------------------------|
  | 1.0   | Full pie (default)           |
  | 0.4   | Donut (60% hole in center)   |                                                                       
  | 0.1   | Very thin ring               |
                                                                                                                 
  ---             
  Why no spines/grid cleanup here?                                                                               
                                                                                                                 
  Notice Chunk 3 had this whole block:
  ax_tl.spines['top'].set_visible(False)                                                                         
  ax_tl.grid(axis='y', ...)             
  ax_tl.set_axisbelow(True)
                                                                                                                 
  Chunk 4 has none of that. Because pie/donut charts use polar coordinates, not Cartesian. There is no x-axis, no
   y-axis, no gridlines — the coordinate system doesn't exist on a pie chart. Calling .spines or .grid on a pie  
  axis would do nothing visible anyway.
                                                                                                                 
  ---             
  Title uses total_transactions
                                                                                                                 
  ax_tr.set_title(f'Clean vs Flagged ({total_transactions} LCs)')

============== Chunk 5 — Fill axes[1, 0]: Severity breakdown bars ===========

Bottom-left panel. The "how URGENT is the backlog?" chart. Same pattern
as the validator Pareto, but bars colored by SEVERITY values (since the
X-axis IS severity, color = X-axis = redundant but reinforcing).

This is the chart where SEVERITY_COLORS shines as a dict — we just look
up each X-axis label in the dict.

Decoding the color list comprehension:

| Piece                                          | What it does               |
|------------------------------------------------|----------------------------|
| [SEVERITY_COLORS[sev] for sev in ...index]     | Build list with same order |
| severity_counts.index                          | Iterate over X-axis labels |
| SEVERITY_COLORS[sev]                           | Lookup color for each      |

This is the SAME pattern as Day 66 with bar_colors, but here it's cleaner
because SEVERITY_COLORS keys ARE the severity strings — direct lookup,
no reindex needed.

Why no annotation here? Because the X-axis labels ARE the story. CRITICAL
written under a bar of size 45 is already telling you "45 critical errors."
An annotation like "45 must-fix-now" would be redundant. Save annotations
for charts where the insight isn't obvious from the labels.

Pro tip — when NOT to add annotations:

| Chart shows                            | Annotation needed?   |
|----------------------------------------|----------------------|
| Pareto with one dominant category      | YES (highlight #1)   |
| Severity with named X-axis labels      | NO (self-explaining) |
| Time series with clear peak/trough     | YES (point out)      |
| Comparison of A vs B (donut)           | NO (chart says it)   |
| Geographic with one outlier country    | YES (call out US)    |

Rule: annotate when the insight isn't OBVIOUS from labels alone. If the
chart already tells the story, more text is noise.

Banking dashboards typically have 1-3 annotations per dashboard, NOT 1
per chart. Pick the most important findings. Less is more.

============== Chunk 6 — Fill axes[1, 1]: Top countries horizontal bars + 2nd annotation ===========

Decoding the new techniques:

| Piece                                        | What it does                  |
|----------------------------------------------|-------------------------------|
| flagged_by_country.idxmax()                  | Returns the INDEX of the max  |
|                                              | (the country code, not count) |
| flagged_by_country.max()                     | Returns the count itself      |
| top_country_pct                              | Compute % dynamically         |
| xy=(count, country_code)                     | Arrow tip at (X=count, Y=code)|
|                                              | NOTE: X and Y SWAPPED for barh|
| xytext=(count * 0.45, 'CH')                  | Text in middle empty space    |
| ax_br.grid(axis='x', ...)                    | Grid on X for HORIZONTAL bars |
|                                              | (X carries the values now)    |

The xy/xytext gotcha for horizontal bars:
- For .bar() (vertical):   xy=(category_label, value)
- For .barh() (horizontal): xy=(value, category_label)

The X/Y axes flip when you flip the chart. Always check which axis carries
the VALUE — that's the one xy uses for the value coordinate.

Pro tip — placing annotations on horizontal bars

The xytext coordinate (count * 0.45, 'CH') is doing two things:

1. count * 0.45 → "place text at 45% of the way along the X-axis"
   Using a FRACTION of the max ensures the text stays inside the chart
   even if the max changes. If US becomes 100, xytext goes to 45. If
   US becomes 200, xytext goes to 90. Self-scaling annotation. ✅

2. 'CH' → "vertically align with the CH country bar"
   Using a category from the Y-axis is a smart trick. CH is roughly
   middle of the country list, so the text sits mid-chart. If you wanted
   text higher, use 'JP' or 'KR'. Lower, use 'GB' or 'TW'.

Both coordinates use REAL data references, not raw pixel positions.
This is "self-scaling annotation" — works no matter how data changes.

Note on the 2nd annotation:

This annotation pattern has a banking name: "actionable insight callout."
It doesn't just describe what the chart shows ("US is the biggest country").
It tells the reader WHAT TO DO ABOUT IT ("← train branch ops").

Two-annotation rule for dashboards:
- 1st annotation: the big finding (LEI dominates)
- 2nd annotation: the actionable insight (where to look)

Each chart-level insight earns its keep by suggesting a next step. That's
the Chief Risk Officer's question every time: "OK, so what do we DO?"
The annotation answers that for them BEFORE they ask.


The complete dashboard story, read in 10 seconds:

| Panel       | What it tells you                                   |
|-------------|-----------------------------------------------------|
| Top-left    | LEI dominates pipeline errors (39%, root cause)    |
| Top-right   | 64% of LCs are flagged — major data quality issue  |
| Bottom-left | Most errors are HIGH severity (158/237 = 67%)      |
| Bottom-right| US is the dominant source (31%, train branch ops)  |

Read together: "Of 200 LCs, 64% are flagged. Of 237 errors, most are HIGH
severity, dominated by LEI issues, and the US branch is the largest
contributor. Action: investigate LEI data source AND train US branch ops."

A Chief Risk Officer reading this gets the WHAT, the HOW BAD, the WHY,
and the WHAT TO DO, all in 10 seconds. THAT is the goal of dashboards.

  | Horizontal  | (40, 'US')      | (value, category_name)      |                                                
                                                                                                                 
  On a horizontal bar chart, X holds the values, Y holds the categories. So the axes are flipped from everything 
  you've done before:                                                                                            
                                                                                                                 
  xy=(top_country_count, top_country),       # (40, 'US')   — tip of US bar                                    
  xytext=(top_country_count * 0.45, 'CH'),   # (18, 'CH')   — text in empty middle                               
                                                                                                                 
  The * 0.45 is a judgment call — it lands the text at ~45% of the max bar length, which happens to be empty     
  space. No formula, just "somewhere that doesn't overlap anything."                                             
                                                                                                                 
  ---                                                                                                          
  Grid axis flip

  ax_br.grid(axis='x', alpha=0.3, ...)  # Vertical lines
  ax_br.grid(axis='y', visible=False)   # No horizontal lines                                                    
   
  Vertical bars → values on Y → horizontal grid lines (axis='y').                                                
  Horizontal bars → values on X → vertical grid lines (axis='x').                                              
                                                                                                                 
  The rule: grid lines always follow where the values are. If you left it as axis='y' here, you'd get horizontal 
  lines crossing the country labels — completely useless. This is the one line most people copy-paste wrong when
  switching from .bar to .barh.

===================== Chunk 7 — Export to PDF + PNG (the final move) ================

Why TWO formats? Different downstream uses:

| Format | Best for                              | Size       |
|--------|---------------------------------------|------------|
| PDF    | Board decks, regulatory PDFs, print   | ~30-100 KB |
| PNG    | Email, slides, GitHub README, web     | ~300-800 KB|

PDF is vector — opens in Acrobat/browser, prints crisp at any size.
PNG is raster — ready to embed anywhere images go.

The naming convention 'lc_dashboard_v1' is intentional. When you iterate
on this dashboard later, you'll save v2, v3, etc. Versioning protects
older approved versions from accidental overwrites. Banks keep every
version of every published chart for 7 years (regulatory retention).
